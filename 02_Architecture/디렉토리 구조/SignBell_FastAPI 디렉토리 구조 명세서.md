# SignBell FastAPI 서버 디렉토리 구조

## 1. 개요

본 문서는 SignBell 프로젝트의 **FastAPI 서버** 부분 디렉토리 구조를 설명합니다. 주로 ML 모델 서빙 및 WebRTC 시그널링 처리를 담당합니다. 각 디렉토리의 역할 중심으로 기술하여 프로젝트 구조 이해를 돕고 코드 관리의 일관성을 유지하는 것을 목표로 합니다.

* **AI 모델러**: 학습된 모델의 서빙 환경 구조를 이해하고 모델 배포 및 업데이트 시 참고
* **신규 합류자**: FastAPI 서버의 역할과 코드 구조를 빠르게 파악


## 2. 최상위 디렉토리 구조

```

signbell-fastapi/
├── .github/    \# GitHub 관련 설정 (워크플로우, 템플릿)
├── certs/      \# SSL/TLS 인증서 관련 파일
├── configs/    \# 서버 설정 관리
├── data/       \# 데이터 저장 관련 (개인 학습 데이터 등)
├── docs/       \# 프로젝트 관련 추가 문서
├── models/     \# ML 모델 파일 (\*.pth)
├── processing/ \# 데이터 처리 및 예측 관련 모듈
├── routers/    \# API 라우터 정의
├── scripts/    \# 유틸리티 스크립트
├── security/   \# 보안 관련 (JWT 검증 등)
└── storage/    \# 데이터 저장/로드 로직 (S3, 로컬 파일 등)

```

* `.github/`: GitHub Actions 워크플로우, 이슈/PR 템플릿 등 GitHub 관련 설정 파일 위치.
* `certs/`: HTTPS 통신을 위한 SSL/TLS 인증서 파일 및 관련 가이드 문서 위치.
* `configs/`: 환경 변수 로드 등 서버의 설정을 관리하는 Python 파일 위치.
* `data/`: 사용자가 업로드한 개인 학습 데이터 등 서버에서 관리하는 데이터 파일 위치 (혹은 마운트 지점).
* `docs/`: FastAPI 모듈화 가이드, 연동 방법 등 개발 관련 보조 문서 위치.
* `models/`: 학습 완료된 PyTorch 모델 파일(`*.pth`) 위치.
* `processing/`: MediaPipe 랜드마크 추출, 모델 예측 수행 등 핵심 데이터 처리 로직 모듈 위치.
* `routers/`: API 엔드포인트(경로)를 기능별로 정의하는 파일 위치. `main.py`에서 이 라우터들을 포함.
* `scripts/`: 개발 편의성을 위한 보조 스크립트 (예: 스토리지 테스트) 위치.
* `security/`: JWT 토큰 유효성 검사 등 보안 관련 로직 모듈 위치.
* `storage/`: 개인 학습 데이터 등을 로컬 파일 시스템 또는 AWS S3와 같은 외부 스토리지에 저장하고 로드하는 로직 모듈 위치.
