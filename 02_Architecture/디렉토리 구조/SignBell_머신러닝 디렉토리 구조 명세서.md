# SignBell 머신러닝(ML) 디렉토리 구조

## 1. 개요

## 2. 최상위 디렉토리 구조

```

signbell-ml/
├── coordinates\_output/ \# 추출된 좌표 데이터 및 메타데이터
├── data\_preparation/   \# 데이터 전처리 스크립트
├── models/             \# 학습된 모델 및 결과 파일
└── test\_results/       \# 모델 테스트 결과 저장

```

* `coordinates_output/`: 원본 영상 데이터에서 추출된 랜드마크 좌표 데이터(`.npy`)와 각 데이터에 대한 정보(파일명, 레이블 등)를 담은 메타데이터(`.csv`) 파일 위치. 모델 학습의 입력 데이터.
* `data_preparation/`: 원본 영상에서 랜드마크 좌표를 추출하는 등 모델 학습에 필요한 데이터를 준비하는 Python 스크립트 위치.
* `models/`: 모델 학습 후 생성된 결과물들이 저장되는 위치. 학습된 모델 가중치 파일 (`*.pth`), 학습 과정 기록 (`*history.csv`), 성능 리포트 (`classification_report.csv`), 혼동 행렬 이미지 (`confusion_matrix.png`) 등 포함.
* `test_results/`: 별도의 테스트 데이터셋으로 모델 성능을 평가한 후, 그 결과를 저장하는 위치 (예: 각 테스트 샘플별 예측 결과).
