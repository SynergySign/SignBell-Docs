#  수어 인식 모델 개발 과정 요약

팀원 [강관주](https://github.com/Kanggwanju)의 수어 인식 모델 개발 과정은 초기 낮은 성능과 심각한 과적합 문제를 해결하기 위해 **모델 복잡도 조정** 및 **데이터 증강/검증 전략 개선**에 초점을 맞추고 진행되었습니다.

### 1단계: 초기 모델 및 문제점 발견 (v1)

| 특징 | 내용 | 결과/문제점 |
| :--- | :--- | :--- |
| **모델** | 단순 LSTM 2층 (64, 32 유닛) | **매우 낮은 성능 (테스트 정확도 52%)** |
| **데이터** | 40배 과도한 데이터 증강 → 10배로 감소 | 증강 축소 후 성능 40%로 악화, 학습 불안정 |
| **진단** | 모델이 유의미한 패턴을 학습 못함, **원본 데이터(111개)가 너무 적음** |

---

### 2단계: 복잡한 모델 도입 및 심각한 과적합 발생 (v2)

| 특징 | 내용 | 결과/문제점 |
| :--- | :--- | :--- |
| **모델** | **Bidirectional LSTM + Attention** 도입 (복잡도 증가: 438K 파라미터) | 복잡한 모델로 패턴 파악 시도 |
| **데이터** | 15배 증강, 4가지 개선된 증강 기법 적용 | **치명적인 실수:** Train/Test 분리 전 증강으로 **데이터 유출** 발생 |
| **진단** | **테스트 정확도 100% (과적합/암기)**. 실제 정확도 60%로 실망스러운 결과. 올바른 데이터 분리/증강 전략 필요성 확인. |

---

### 3단계: 올바른 데이터 전략과 모델 단순화 (v3)

| 특징 | 내용 | 결과/문제점 |
| :--- | :--- | :--- |
| **모델** | **Attention 제거** 및 구조 단순화 (135K 파라미터), Bidirectional LSTM 2층 | 모델 복잡도를 데이터 양에 맞게 조정 |
| **데이터** | **✅ 올바른 전략:** **Train/Test 분리 후 Train만 5배 증강** | 훈련은 성공적으로 수행되었으나 (Test 정확도 100% 도달) |
| **진단** | **Test 데이터(23개)가 너무 적어** 100% 정확도도 신뢰하기 어려움. **실제 영상 테스트 40%**로 성능 저조 (특히 '일요일', '미국', '월세' 오분류). **'무한' 편향** 문제 발견. |
| **해결 방안** | **직접 수어 영상 촬영**을 통한 원본 데이터 셋 확대 (최소 50개 $\to$ 권장 100개) 제안 |

---

### 4단계: 최종 전략 - K-Fold 교차 검증 및 정규화 강화 (v5)

| 특징 | 내용 | 최종 결과 |
| :--- | :--- | :--- |
| **모델 구조** | v3의 Bidirectional LSTM 구조 기반, **L2 정규화** 추가 및 **Dropout 비율 상향 (0.5 → 0.6)** | 과적합 억제 및 일반화 성능 강화 |
| **평가 전략** | **✅ K-Fold 교차 검증 도입** (StratifiedKFold) | 단일 Test 셋의 낮은 신뢰도를 극복하고 모델 일반화 성능 신뢰도를 대폭 향상 |
| **최종 평가** | **평균 테스트 정확도: 82.02% (± 3.86%)** | **학습 단계에서는 매우 훌륭한 성능 확보** (낮은 표준편차로 일반화 성능 우수) |
| **실제 예측** | **실제 정확도: 40% (2/5)** | **여전히 실제 영상 테스트에서 성능 저조** ('수학', '미국' 제외 모두 오분류). '미국'과 유사성 문제 심각. |
| **결론** | **모델 학습 방법론은 성공적**이나, **'일요일', '무한', '월세'**에 대한 훈련 데이터의 **양과 질 문제** (다른 단어와의 유사성)가 가장 큰 병목으로 재확인됨. |

---

##  개발 과정으로부터 얻은 주요 교훈

1.  **원본 데이터의 중요성:** 모델의 잠재력에도 불구하고, **원본 데이터 111개**의 절대적인 부족함과 특정 클래스 ('일요일', '무한', '월세')의 데이터 품질/특징 부족이 실제 성능 저하의 가장 큰 원인입니다.

2.  **데이터 전략의 영향력:** v2의 데이터 유출 문제는 극단적인 과적합을 유발했으며, v3와 v5에서 **Train/Test 분리 후 증강** 및 **K-Fold 교차 검증**을 통해 신뢰할 수 있는 학습 환경을 구축했습니다.

3.  **일반화 성능과 현실 세계의 격차:** K-Fold에서 82%의 높은 평균 정확도를 달성했음에도 불구하고, 직접 촬영한 영상에서 40%의 낮은 정확도를 보인 것은 **훈련 데이터셋과 실제 사용 환경의 데이터 분포에 큰 차이**가 있음을 명확히 보여줍니다.
