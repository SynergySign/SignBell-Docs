# 수어 인식 모델 개발 과정 요약

팀원 [고동현](https://github.com/rhehdgus8831)의 수어인식 모델 개발 과정은 **높은 검증 정확도(84%)**를 확보했음에도 불구하고 **일반화 실패**라는 근본적인 문제에 직면하여, **'모델 중심'에서 '데이터 중심'으로 접근 방식을 전환**하는 데 초점을 맞추고 진행되었습니다.

---

### 1단계: 모델 튜닝 및 '가짜 정확도' 확보

| 특징 | 내용 | 결과/문제점 |
| :--- | :--- | :--- |
| **모델** | **Transformer 모델** 기반으로 데이터 증강, 모델 경량화, 3D 좌표 변환 등 기술적 튜닝 적용. | **최고 검증 정확도 84% 달성** 및 일부 테스트 영상에서 정답을 맞히는 성능 개선 가능성 확인. |
| **데이터** | 3D 좌표 추출 방식 적용 및 7개 단어(기쁨, 미국, 수학, 안녕, 월세, 인사, 일요일) 데이터셋 사용. | **일반화(Generalization) 실패** 문제 발생. 새로운 테스트 영상에서 특정 단어('미국', '인사' 등)로만 예측하는 현상 지속. |
| **진단** | **기술적 문제 (3D 좌표 오류)**: `ValueError` 오류 발생. 2D 벡터를 3D 좌표 정규화 기준점으로 잘못 설정한 코드 문제. | 3D 벡터 `np.array([0.0, 0.0, 0.0])`로 수정하여 **데이터 처리 파이프라인의 문제는 해결**했음을 확인. |

---

### 2단계: 근본 원인 분석 및 전략 전환

| 특징 | 내용 | 결과/문제점 |
| :--- | :--- | :--- |
| **원인 분석** | **데이터의 절대적인 양과 다양성 부족**이 가장 큰 원인. | 모델이 수어 동작 패턴이 아닌, 특정인의 손 모양, 촬영 각도 등 **피상적인 특징에 과적합(Overfitting)**된 상태로 진단. |
| **문제 정의** | 84%의 정확도는 학습 데이터와 유사한 패턴에만 국한된 **'가짜 정확도'**로 정의. 모델이 익숙한 '연습문제'만 잘 푸는 상태였음. | 높은 정확도에 대한 **좌절감**과 **코드 수정만으론 한계**가 있다는 중요한 교훈을 얻음. |
| **해결 방안 결정** | 접근 방식을 **'모델 중심'에서 '데이터 중심'으로 전환**하기로 결정. | 단순히 테스트 반복 대신 **'오답노트(Confusion Matrix)'**를 작성하여 모델의 취약점 분석. |
| **최종 액션** | 분석된 취약점을 바탕으로, **자주 혼동하는 단어들을 위주**로 **다양한 사람과 환경**에서 촬영한 데이터를 추가 수집하는 것을 최종 해결책으로 결정. |

---

##  개발 과정으로부터 얻은 주요 교훈

1.  **'가짜 정확도'의 허무함:** 84%라는 높은 검증 정확도가 실제 테스트 성능과 불일치하는 **일반화의 벽**에 부딪히며, 딥러닝 프로젝트에서 **검증 정확도라는 숫자가 얼마나 허무할 수 있는지** 뼈저리게 깨달았습니다.

2.  **데이터 중심(Data-centric)의 중요성:** 코드를 수정하고 파라미터를 조정하는 기술적 노력만으로는 한계가 있음을 확인하고, 문제의 근본 원인이 **'코드'가 아닌 '데이터'**에 있다는 것을 스스로 증명하여 **접근 방식을 전환**하는 중요한 이정표를 세웠습니다.

3.  **오류 분석의 체계화:** 무작정 개발하는 대신 **'오답노트(Confusion Matrix)'**와 같은 체계적인 분석을 통해 모델의 취약점을 명확히 파악하는 것이, 앞으로의 프로젝트 방향을 설정하는 데 가장 중요하다는 것을 깨달았습니다.
