## 논의 내용
### **1. 팀원별 진행 상황 (전일 작업 내용)**
* **강관주:**
  * **다중 탭 접속 제한 기능 구현:** 유저가 하나의 탭으로만 플랫폼을 이용하도록 퇴장 로직 개발.
  * 백엔드에서 세션을 생성하고 관리하여, 이미 세션이 있는 경우 새로운 연결을 막는 방식으로 구현.
  * 프론트엔드와 연결하여 추후 테스트 예정.

* **백승현:**
  * **국어원 API 관리 테이블 분리:** API 호출 및 데이터 관리를 위한 테이블을 별도로 분리하여 구조 개선.
  * **RTC 좌표 데이터 저장 계획:** RTC 기능 개발이 완료되는 대로, 추출된 좌표 데이터를 S3에 저장하는 기능 구현 예정.

* **고동현:**
  * **WebRTC 동시 접속 테스트 완료:** Janus 서버를 통해 4명 동시 입장을 테스트하고, 관련 코드를 병합 완료.

### **2. 신규 기능 및 개선 사항 논의**
* **개인별 학습 데이터 관리:**
  * **필요성:** 유저별로 학습한 단어, 학습 횟수, 데이터 제공 횟수 등을 추적하고 관리할 필요성 제기.
  * **해결 방안:** 개인 학습 데이터를 저장하기 위한 별도의 DB 테이블을 추가하기로 논의.
* **'기타' 카테고리 UI 개선:**
  * **문제점:** 현재 카테고리 내 항목이 많아 무한 스크롤 방식이 사용자 경험에 비효율적일 수 있다는 의견.
  * **개선 방안:** 무한 스크롤 대신 **페이지네이션(Pagination)**을 도입하여 가독성과 성능을 개선하는 방안 제안.
