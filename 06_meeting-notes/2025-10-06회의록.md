## 논의 내용

### 1. 모델별 테스트 결과 공유

각 팀원이 5개의 테스트 영상으로 모델 성능을 검증한 결과를 공유했으며, 유의미한 결과는 도출되지 않았습니다.

* **강관주**
    * **테스트 모델**: 5가지 모델 (LSTM, Bidirectional LSTM + Attention, Bidirectional LSTM, CNN + Bidirectional LSTM, Bidirectional LSTM + K-Fold 교차 검증) 및 데이터 증강 방식 변경을 시도함.
    * **결과**:
        * 단순 LSTM: 학습 정확도 낮음.
        * 3가지 모델: 학습 정확도가 1에 가까운 **과적합(Overfitting)** 문제 발생. 모델이 훈련 데이터를 암기해버림.
        * `Bidirectional LSTM + K-Fold 교차 검증`: 평균 학습 정확도는 82.02%로 과적합은 피한 것으로 보이나, 실제 테스트 정확도는 40%(5개 중 2개 정답)로 매우 낮게 나옴.

* **백승현**
    * **테스트 모델**:
        1.  기존 `Bi-LSTM + Attention (TripletLoss 방식)` → `Bi-LSTM + 1D CNN + Attention (Cross-Entropy Loss 방식)`으로 변경.
        2.  `동작 변화점 탐지 + Multi-Head Attention` 방식 시도.
        3.  좌표 추출 방식을 기존 2D에서 **3D로 변경** 후 테스트.
        4.  좌표에 델타 데이터(손의 이동 속도)를 추가.
    * **결과**:
        * 특징적인 첫 동작은 감지하나, 이후 유사한 동작들을 구분하지 못함 (가중치 문제).
        * Multi-Head Attention 방식도 여전히 불안정함.
        * 델타 데이터 추가는 모델을 무겁게만 만들고 성능 향상에 기여하지 못함.

* **고동현**
    * **테스트 모델**: 영상 전체 동작의 관계를 한 번에 파악하는 데 유리한 **Transformer** 모델 사용.
    * **결과**:
        * 초기 정확도가 매우 낮고, 특정 단어('미국')만 예측하는 등 일반화에 실패함.
        * 데이터 증강, 모델 최적화 등 기술적 튜닝으로 정확도를 76%까지 올렸으나, 이는 모델이 동작이 아닌 배경 등 **엉뚱한 정보를 암기한 '가짜 정확도'**로 판명됨.
        * MediaPipe로 배경을 제거하고 순수 동작 데이터로만 학습시키자 정확도가 30%대로 하락.
        * **결론**: 현재 보유한 **절대적인 데이터 양 부족**이 근본적인 한계이며, 현재 데이터로는 어떤 모델을 사용해도 일반화 성능 확보가 어렵다는 것을 체계적으로 증명함.

## 결정 사항
1.  **(데이터셋 변경)**: 현재 모델들이 공통적으로 '무한'이라는 단어를 제대로 학습하지 못하는 문제가 있어, 해당 단어를 데이터셋에서 제외하기로 함.
2.  **(신규 데이터 추가)**: '무한'을 대체할 새로운 단어로 **'기쁨', '인사', '안녕'** 3개를 선정하고, 각 단어별로 5개 이상의 영상을 촬영하여 데이터셋을 보강하기로 함. (총 7개 단어로 테스트 예정)
3.  **(데이터 추출 방식 통일)**: 백승현님이 시도하여 공유한 **3D 좌표 추출 방식**을 모두의 모델에 공통적으로 적용하여 테스트하기로 함.
