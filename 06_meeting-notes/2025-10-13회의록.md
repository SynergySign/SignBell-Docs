## 논의 내용
- **개발 일정:** 전체적인 개발 마일스톤(백엔드 ~10/20, 프론트엔드 ~10/27)에 대해 논의했으며, 세부 일정은 10월 14일 회의에서 확정하기로 했습니다.
- **Web RTC 테스트:** 개발 중 Web RTC 기능 테스트 시, 기본 코드와 분리를 위해 테스트용 컴포넌트 및 파일명에 `_test` 접미사를 붙이는 규칙을 논의했습니다.
- **좌우 반전:** 사용자 환경에 따라 일관된 경험을 제공하기 위해, 로직을 통한 좌우 반전 처리 또는 MediaPipe 라이브러리의 자동 추적 기능 활용 가능성을 검토했습니다.
- **개인 수어 학습 기능 흐름:** 단어 선택부터 데이터 수집까지의 사용자 플로우(모달 UI, 거울 모드, 카운트다운, 프레임 수집)를 구체적으로 정의했습니다.
- **퀴즈 기능:** 실시간 상호작용을 위해 Web Socket 도입과 퀴즈 데이터 조회 방식에 대한 기술적인 논의를 진행했습니다.
- **데이터 수집 보상:** 모델에 없는 새로운 단어 데이터를 제공하는 사용자에 대한 동기 부여 방안으로 프로필 테두리, 뱃지 등 비재화적 보상 시스템에 대해 논의했습니다.
- **모델 버전 관리:** 수집된 데이터를 바탕으로 AI 모델을 업데이트하는 주기를 정해야 한다는 점에 대해 논의했습니다.
- **GitHub 라벨:** 프로젝트 관리 효율화를 위해 GitHub Actions를 이용하여 Organization 전체에 일관된 라벨을 적용하는 방안을 논의했습니다.
- **로그 관리:** 서비스 안정성 확보를 위해, 개발 여유가 된다면 배포 환경에서 발생하는 서버 로그를 저장할 별도의 테이블 구축을 고려하기로 했습니다.

## 결정 사항
- **개발 마일스톤:** 1차 백엔드 개발 완료(~10/20), 1차 프론트엔드 개발(~10/27), 배포 완료(~10/27 이후)를 목표로 합니다. (상세 일정은 10/14 확정)
- **Web RTC 테스트 규칙:** 테스트용 파일명 뒤에 `_test`를 붙여 생성합니다.
- **개인 수어 학습 흐름:** 논의된 UI/UX 흐름(단어리스트 → 모달 → 데이터 제공 페이지)을 기준으로 개발을 진행합니다.
- **데이터 제공 보상:** 신규 단어 데이터 제공 시 프로필 테두리, 뱃지, 칭호 등의 보상을 지급합니다.
- **GitHub 라벨 관리:** GitHub Actions를 활용하여 전역 라벨을 추가합니다.
