## SignBell 비기능 요구사항 명세서

본 문서는 SignBell 서비스의 비기능 요구사항(Non-functional Requirements, NFR)을 정의하여 시스템의 품질 속성, 제약 조건, 성능 목표를 명확히 합니다.

* **작성자**: [송민재](https://github.com/songkey06)
* **작성일**: 2025-10-30
* **문서 버전**: v1.0.0
* **기반 문서**: SignBell 프로젝트 개요, 사용자 흐름도, 기능 우선 순위

**대상 독자:**

* **기획자 / PM**: SignBell의 기능 범위, 우선순위, 프로젝트 일정 및 리소스를 관리하는 담당자. 시스템의 **품질 속성(성능, 보안, 가용성)** 및 제약 조건을 이해하고 기획 방향성을 검토하는 담당자.
* **백엔드 개발자**: WebRTC 시그널링 서버, AI 모델 서빙 API, 실시간 퀴즈 로직 등을 구현·유지보수하는 개발자. 특히 **성능 요구사항(응답 시간, 처리량)** 및 **보안 요구사항(인증, 암호화)**을 만족시키기 위해 참조해야 합니다.
* **프론트엔드 개발자**: 실시간 캠 영상 처리, WebRTC 기반 화상 퀴즈 UI, 사용자 인터페이스 기능을 설계·구현하는 개발자. **사용성 및 성능 요구사항**을 충족시키기 위해 참조해야 합니다.
* **DevOps / 인프라 엔지니어**: WebRTC 서버 및 AI 모델 서빙 환경 구축, CI/CD 파이프라인 운영, **보안 설정 및 시스템의 가용성**을 관리하는 담당자.
* **QA / 테스트 엔지니어**: 사용자 시나리오별 테스트 케이스 작성, AI 모델 정확도 검증 등 **비기능 요구사항에 기반한 테스트**를 수행하는 담당자.
* **AI 모델러**: AI 모델의 **정확도 및 성능(응답 속도)** 요구사항을 목표치에 맞게 개발 및 최적화하는 담당자.
* **디자이너 (UX/UI)**: 기능 및 사용자 흐름에 기반하여 화면 설계와 사용자 경험(UX)을 구체화하는 담당자. 특히 **사용성 및 접근성 요구사항**을 검토해야 합니다.

---

## 1. 개요

SignBell 프로젝트는 AI 기반 실시간 수어 학습 플랫폼으로, **성능(AI 응답 속도, WebRTC 안정성)**과 **보안(데이터 수집, 통신 암호화)**이 서비스의 핵심 성공 요소입니다.

---

## 2. 비기능 요구사항 상세 명세

요구사항은 **'NFR-(분류)-(번호)'** 형식으로 ID를 부여하며, 주요 분류는 성능, 보안, 가용성, 유지보수성, 사용성/접근성으로 구분합니다.

## 3. 제약 사항

* **비즈니스 제약:** 서비스의 범위 및 정책에 대한 비즈니스 제약 사항은 [SignBell 기능 요구사항 명세서 - 5. 제약 사항](https://github.com/SynergySign/SignBell-Docs/blob/dev/01_Planning/SignBell_%EA%B8%B0%EB%8A%A5%20%EC%9A%94%EA%B5%AC%EC%82%AC%ED%95%AD%20%EB%AA%85%EC%84%B8%EC%84%9C.md)을 참조하십시오.
* **기술적 제약:**
    * AI 모델 서빙을 위해 Python 기반의 **FastAPI** 사용을 고수해야 합니다. (NFR-REL-01 관련)
    * WebRTC 통신을 위한 시그널링 서버 구성이 필수입니다. (NFR-PER-02 관련)
    * 배포 환경은 **CI/CD 파이프라인**을 통해 자동화되어야 합니다. (NFR-MNT-03 관련)
* **보안/법적 제약:**
    * 모든 사용자 통신은 **HTTPS/WSS**를 통해 암호화되어야 합니다. (NFR-SEC-01 관련)
    * 사용자 영상은 어떠한 경우에도 저장할 수 없습니다. (NFR-SEC-03 관련)

## 4. 용어 정의 (Glossary)

| 용어                     | 정의                                                               |
|:-----------------------|:-----------------------------------------------------------------|
| **한국수어(KSL)**          | Korean Sign Language, 대한민국의 농인 사회에서 사용되는 고유한 수화 언어               |
| **거울 모드(Mirror Mode)** | 웹캠을 통해 사용자의 동작을 실시간으로 미러링하여 보여주는 학습 모드                           |
| **MediaPipe**          | Google이 개발한 오픈소스 프레임워크로, 실시간으로 사람의 포즈, 손, 얼굴 등의 좌표를 추출하는 라이브러리   |
| **WebRTC**             | Web Real-Time Communication, 웹 브라우저 간 실시간 음성 및 영상 통신을 가능하게 하는 기술 |
| **방장(Host)**           | 퀴즈 방을 생성한 사용자로, 퀴즈 시작 권한을 가짐                                     |
| **유사도 검사**             | AI 모델이 사용자의 수어 동작과 정답 동작의 유사도를 계산하여 정답 여부를 판정하는 과정               |
| **좌표 데이터**             | MediaPipe로 추출한 신체 관절 포인트의 X, Y, Z 좌표 값                           |
| **액세스 토큰**             | 사용자 인증 상태를 확인하기 위한 단기 유효 토큰                                      |
| **리프레시 토큰**            | 액세스 토큰을 재발급하기 위한 장기 유효 토큰                                        |
