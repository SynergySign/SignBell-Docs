# SignBell 개발 일정 및 마일스톤

본 문서는 SignBell 프로젝트의 기획 단계부터 최종 발표까지의 전체 개발 일정과 주요 마일스톤을 정의합니다.


* **작성자:** [고동현](https://github.com/rhehdgus8831)
* **작성일**: 2025-10-27
* **최종 수정일**: 2025-10-27
* **문서 버전**: v1.0.0

**대상 독자:**

* **팀원 전원**: 개발 일정 및 마일스톤 확인, 개인별 작업 계획 수립
* **팀장/PM**: 프로젝트 진행 상황 관리 및 일정 조정
* **신규 합류자**: 프로젝트 전체 일정 및 개발 단계를 빠르게 파악

---

## 1. 개요

SignBell 프로젝트는 AI 기반 실시간 수어 학습 플랫폼으로, MVP(Minimum Viable Product) 개발 및 배포를 목표로 합니다. 본 문서는 2025년 9월 29일부터 11월 3일까지 약 5주간 진행된 프로젝트의 전체 일정을 기록하고, 주요 단계별 목표(마일스톤)를 관리하기 위해 작성되었습니다.

---

## 2. 주요 마일스톤

프로젝트 전체 일정을 주요 달성 목표(마일스톤) 기준으로 요약한 표입니다.

| 마일스톤                            | 기간 | 주요 목표 |
|:--------------------------------| :--- | :--- |
| **1. 기획 및 설계**                  | 9월 29일 ~ 10월 3일 | 프로젝트 주제 선정, MVP 정의, 핵심 문서 작성 (요구사항, 유저플로우, DB/아키텍처 설계), ML 리서치 |
| **2. 초기 개발 (AI/ML/Frontend)**   | 10월 4일 ~ 10월 12일 | AI 모델 개발 착수, 학습 데이터 수집, 프론트엔드 와이어프레임 및 기본 UI 작업 |
| **3. 핵심 기능 개발 (Backend/AI/ML)** | 10월 13일 ~ 10월 19일 | 백엔드 API (WebSocket, REST, JWT 인증) 개발, 국립국어원 API 연동, ML 모델 개발 지속 |
| **4. 기능 고도화 (Frontend)**        | 10월 20일 ~ 10월 28일 | Janus WebRTC 연동, 백엔드 API 연동(fetch), 전반적인 UI/UX 보강 |
| **5. 통합, 배포 및 테스트**             | 10월 29일 ~ 10월 30일 | 전 파트(FE/BE/AI) 기능 통합 테스트, 발견 버그 수정, AWS 배포, 문서 최신화 |
| **6. 안정화 및 발표 준비**              | 10월 31일 ~ 11월 2일 | 배포 환경에서의 최종 버그 픽스 및 서비스 안정화, PPT 등 발표 자료 준비 |
| **7. 최종 발표**                    | 11월 3일 | 프로젝트 결과물 최종 발표 |

## 3. 세부 개발 일정

각 마일스톤별 세부 진행 내역입니다.

### 1단계: 기획 및 설계 (9월 29일 ~ 10월 3일)

* **목표**: 프로젝트의 방향성 확립 및 구체적인 계획 수립
* **주요 활동**:
    * 프로젝트 주제(AI 수어 학습) 선정 및 MVP 범위 논의
    * 구현 가능성 및 기술 스택(Spring, React, FastAPI, WebRTC) 검토
    * 주요 산출물(문서) 작성:
        * 프로젝트 개요
        * 배경 및 시장조사 명세서
        * 사용자 페르소나 및 유저플로우
        * 기능 요구사항 명세서
    * 백엔드 DB 설계 (ERD 작성) 및 시스템 아키텍처 설계
    * 머신러닝(ML) 관련 자료 리서치 (수어 인식 모델 탐색)

### 2단계: 초기 개발 (10월 4일 ~ 10월 12일)

* **목표**: 각 파트별 기본 토대 마련 및 개발 시작
* **주요 활동**:
    * **FastAPI & ML**:
        * AI 학습 및 추론 기능 개발 시작
        * 모델 학습용 영상 데이터 수집 및 전처리
    * **Frontend**:
        * 와이어프레임 문서화 작업
        * 주요 페이지 UI 디자인 및 퍼블리싱 작업

### 3단계: 핵심 기능 개발 (10월 13일 ~ 10월 19일)

* **목표**: 서비스의 핵심 로직 및 API 구현
* **주요 활동**:
    * **Backend (Spring)**:
        * 실시간 퀴즈방을 위한 웹소켓(WebSocket) API 개발
        * 회원 관리, 퀴즈방 관리 등을 위한 REST API 개발
        * JWT 토큰 기반 인증/인가 기능 구현
        * 국립국어원 Open API 연동 (수어 데이터 확보)
    * **FastAPI & ML**:
        * AI 모델 개발 지속 및 성능 튜닝
    * **Frontend (React)**:
        * 개발된 UI/UX 기능 구현 및 고도화

### 4단계: 기능 고도화 (10월 20일 ~ 10월 28일)

* **목표**: 실시간 통신 기능 구현 및 프론트엔드 완성도 향상
* **주요 활동**:
    * **Frontend (React)**:
        * Janus 미디어 서버를 활용한 WebRTC 연동 (실시간 영상 스트리밍)
        * 백엔드 API 연동 (데이터 `fetch`) 작업
        * 전반적인 UI/UX 보강 및 사용자 경험 개선

### 5단계: 통합, 배포 및 테스트 (10월 29일 ~ 10월 30일)

* **목표**: 개발된 기능의 통합 및 실제 서비스 환경 구축
* **주요 활동**:
    * 전체 기능 통합 테스트 (Frontend ↔ Backend ↔ FastAPI/ML)
    * 통합 테스트 과정에서 발견된 버그 수정
    * AWS(EC2, S3, RDS 등)를 활용한 배포 작업
    * 프로젝트 관련 문서 변경사항 최신화 및 보충

### 6단계: 안정화 및 발표 준비 (10월 31일 ~ 11월 2일)

* **목표**: 서비스 안정성 확보 및 프로젝트 성과 정리
* **주요 활동**:
    * 배포 환경(AWS)에서의 최종 버그 픽스 및 서비스 안정화
    * 프로젝트 최종 발표 자료(PPT) 준비

### 7단계: 최종 발표 (11월 3일)

* **목표**: 프로젝트 결과물 시연 및 발표
* **주요 활동**:
    * SignBell 프로젝트 최종 발표
